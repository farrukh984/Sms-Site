SMS SITE - Complete Project Documentation
Presentation & Architecture Guide

1. Project Overview & Technologies

This application is a real-time messaging, audio, and video calling platform. It is built using modern web development standards to ensure speed, security, and scalability.

  • Backend Framework: ASP.NET Core MVC (net10.0)
  • Database: PostgreSQL (Neon Tech) with Entity Framework Core ORM
  • Real-Time Engine: Microsoft SignalR
  • Media Storage: Cloudinary (for profile pictures and chat attachments)
  • Frontend: HTML5, Vanilla CSS, JavaScript, WebRTC
Samjhane ke liye: Yeh project ASP.NET Core par bana hai. Database PostgreSQL hai jo cloud par hosted hai. Real-time chat aur calling ke liye SignalR aur WebRTC ka istemal kiya gaya hai.

2. Installed Packages (Dependencies)

The following major NuGet packages are installed in the project (visible in Site.csproj):

  • CloudinaryDotNet: Used to upload and retrieve media (images/videos) to the Cloudinary cloud storage so the main server doesn't get overloaded.
  • Npgsql.EntityFrameworkCore.PostgreSQL: The database provider that connects Entity Framework to our PostgreSQL database.
  • Microsoft.AspNetCore.Authentication.Google & Facebook: These packages natively handle the Social Login flow for Google and Facebook without requiring manual API setups.
  • Microsoft.EntityFrameworkCore.Tools: Used for Code-First database migrations.
Samjhane ke liye: Packages ka matlab hai bani banayi libraries. Database se connect hone ke liye Npgsql, tasweerein save karne ke liye Cloudinary, aur Google/Facebook login ke liye inke official packages install kiye gaye hain.

3. Real-Time Chat (Bina Reload Kiye Kaise Chal Raha Hai?)

The core feature of the application is real-time messaging without page reloads. This is achieved using Microsoft SignalR.

How it works under the hood:

Traditional web applications use "HTTP Requests", meaning the browser has to ask the server "Do I have new messages?" and refresh the page. Instead of this, our app uses WebSockets via SignalR.

SignalR establishes a persistent, always-open connection between the user's browser and the server (via ChatHub.cs). When User A sends a message, the server instantly "pushes" this message through the open tunnel to User B. JavaScript catches this push and appends the message to the screen in milliseconds. That is why the page never reloads.

Samjhane ke liye: Chat bina reload kiye SignalR (WebSockets) ki madad se chalti hai. Ye browser aur server ke darmiyan ek open connection bana deta hai. Jaise hi koi message bhejta hai, server khud doosre bande ko message "push" kar deta hai, isliye page refresh nahi karna parta.

4. Voice & Video Calls (Call Ke Liye Kya Use Hua Hai?)

The audio and video calling feature is powered by WebRTC (Web Real-Time Communication).

WebRTC is a powerful browser technology that allows direct Peer-to-Peer (P2P) communication. This means once the call is connected, the video and voice data does not go through our server; it travels directly from User A's internet to User B's internet. This ensures zero lag, high quality, and low server cost.

SignalR's role in calls:

Before WebRTC can connect the two users directly, they need to find each other. We use SignalR as the Signaling Server. It sends the "Ringing" notification, and exchanges network coordinates (called SDP Offers/Answers and ICE Candidates). Once the coordinates are exchanged, WebRTC takes over and the direct call begins.

Samjhane ke liye: Calls ke liye WebRTC use hua hai jo Peer-to-Peer (direct) connection banata hai. Yani data server pe jaye bina direct ek user se doosre user tak jata hai. Call milane (Ringing aur network info share karne) ke liye humne SignalR use kiya hai.

5. Camera & Voice Access (Microphone / Cam Kaise Access Hota Hai?)

To access the user's hardware (Camera and Microphone), the project uses the built-in HTML5 MediaDevices API.

When a user initiates or accepts a call, the JavaScript code executes the following function:

navigator.mediaDevices.getUserMedia({ video: true, audio: true })

How it flows:

  1. The browser intercepts this code and shows a popup asking the user for permission: "Allow Camera and Microphone".
  2. If the user clicks "Allow", the browser hardware generates a MediaStream.
  3. This stream is attached to an HTML <video> tag so the user can see their own face.
  4. Simultaneously, this stream is injected into the WebRTC Peer Connection and transmitted securely to the other user.
Samjhane ke liye: Camera aur Mic access karne ke liye JavaScript ka getUserMedia function chalaya gaya hai. Is se browser user se permission mangta hai. Permission milne ke baad browser ek video/audio stream deta hai jo hum screen par bhi dikhate hain aur doosre bande ko bhi bhej dete hain.

6. Security & Onboarding Flow

The project implements a highly secure onboarding pipeline for new users:

  • OTP Verification: Email verification using dynamic OTPs sent via SMTP to ensure identity.
  • Social Auth: Seamless integration with Google and Facebook OAuth.
  • Multi-Factor Steps: Setup of a 4-digit PIN and Biometric (Fingerprint/FaceID) integration via WebAuthn API for mobile devices.
  • Welcome Notification: A fully styled secure email is dispatched automatically using our custom SMTP service once the user completes the profile onboarding.
Samjhane ke liye: Security ke liye OTP email verification banayi hai. Google/Facebook login bhi hai. User ko apni profile complete karne, 4-digit PIN rakhne aur Fingerprint lagane ka option bhi diya gaya hai. Jab sab complete ho jata hai to usay ek khoobsurat Welcome Email khud-ba-khud chali jati hai.
